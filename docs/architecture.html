多 Agent 研究助手 — LangGraph 编排 + OpenManus Agent 基座

1. 项目概述

多 Agent 协作研究助手。用户输入研究主题，6 个专用 Agent 在 LangGraph 编排下协作完成：

主题分解 → 多源搜索 → 深度分析 → 综合归纳 → 报告撰写 → 质量审查

最终输出：结构化 Markdown 研究报告。

核心设计原则

关注点分离LangGraph 负责编排（何时执行谁），OpenManus 负责 Agent 内部循环（如何思考与行动）
模型可配置每个 Agent 通过 config/agents.yaml 独立配置 provider / model / temperature
质量闭环Critic 评分 → 低于阈值自动重试（最多 3 轮）
复用上游Agent 基座使用 FoundationAgents/OpenManus，不重复造轮子

7. Think → Act 交互流程

sequenceDiagram
    actor User as 用户
    participant Node as LangGraph 节点
    participant Agent as ToolCallAgent
    participant LLM as LLMAdapter
    participant Provider as LLM 提供商
Anthropic/OpenAI/DeepSeek
    participant Tool as ToolCollection
    participant Mem as Memory

    Node->>Agent: run(request)
    activate Agent

    loop ReAct 循环（think → act → observe）
        Agent->>Agent: think()
        Agent->>LLM: ask_tool(messages, system_msgs, tools, tool_choice)
        Note over Agent,LLM: OpenManus 消息格式 → LangChain 消息格式
        LLM->>Provider: ChatModel.bind_tools(tools).ainvoke()
        Provider-->>LLM: AIMessage（content + tool_calls）
        Note over LLM: LangChain AIMessage → OpenManus ToolCallResult
        LLM-->>Agent: ToolCallResult（content + tool_calls[]）

        alt 有工具调用
            Agent->>Agent: act()
            loop 对每个 tool_call
                Agent->>Tool: execute(name, args)
                Tool->>Tool: tool.execute(**kwargs)
                Tool-->>Agent: 结果（字符串或 ToolResult）
                Agent->>Mem: add_message(tool_msg)
            end
            Note over Agent: observe → 下一轮 think()
        else 无工具调用（Terminate）
            Agent->>Agent: _handle_special_tool()
            Note over Agent: state = FINISHED
        end
    end

    deactivate Agent
    Agent-->>Node: 最终响应

8. LLMAdapter — 多 Provider 桥接层

设计动机

OpenManus 的 ToolCallAgent.think() 调用 self.llm.ask_tool()，该接口返回 OpenAI 格式的 ChatCompletionMessage。但 OpenManus 自带的 LLM 类仅支持 OpenAI 兼容 API（通过 AsyncOpenAI 客户端）。

LLMAdapter 包装 LangChain 的 BaseChatModel，对外暴露相同的 ask_tool() 签名，内部使用 LangChain 原生支持 Anthropic / OpenAI / DeepSeek / Ollama。

关键格式转换

转换项OpenManus 格式LangChain 格式
函数参数Function.arguments（JSON 字符串）args（字典）
工具选择ToolChoice.AUTO / REQUIRED / NONE"auto" / "any" / "none"

为何使用 agent.__dict__["llm"] = adapter？ OpenManus 的 BaseAgent 通过 Pydantic model_validator 检查 isinstance(self.llm, LLM)——如果不是 OpenManus 的 LLM 实例，它会创建一个新的并覆盖我们的 adapter。通过 __dict__ 直接赋值可以绕过 Pydantic 验证。

10. LangGraph 工作流

ResearchState 共享状态

class ResearchState(TypedDict, total=False):
    topic: str                         # 研究主题
    language: str                      # 报告语言 zh / en
    outline: list[dict]                # 大纲
    search_queries: list[str]          # 搜索查询列表
    search_results: Annotated[list[dict], operator.add]  # 并行追加搜索结果
    analyses: Annotated[list[dict], operator.add]        # 并行追加分析结果
    synthesized_findings: str          # 综合结论
    draft_report: str                  # 报告草稿
    final_report: str                  # 最终报告
    citations: list[dict]              # 引用列表
    quality_score: float               # 质量评分
    critique: dict                     # 审查意见
    gaps: list[str]                    # 内容缺口
    current_phase: str                 # 当前阶段
    research_round: int                # 当前轮次
    max_rounds: int                    # 最大轮次
    quality_threshold: float           # 质量阈值
